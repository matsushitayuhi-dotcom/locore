# Locore コピー（文言）方針メモ

開発過程で「とりあえず」入れていた仮テキストを、本番想定の編集トーンに書き換えるための備忘。
今後追加する画面は、ここに書いた指針を見ながら書く。

## 声 (voice)

ベース: **小さな旅行誌の編集部** の声。Airbnb の温かさ、Substack のクリエイターファースト感、
Lonely Planet の落ち着いた編集口調を混ぜたイメージ。

| 〇 やる                            | × やらない                                  |
| ---------------------------------- | ------------------------------------------- |
| 「現地で書いている人がいます」     | 「現地クリエイターが本物を届けます」（広告） |
| 「一軒の店、一本の坂道」           | 「珠玉の隠れスポット」（盛り過ぎ）           |
| 「移動時間と注意点まで添えて」     | 「効率的に巡れる！」（マーケ感）             |
| 「ぼくらは長く覚えていたい」       | 「最高のクリエイターを募集中」（採用文言）   |
| 「この街で書ける人を探しています」 | 「クリエイター大募集！」                     |

具体名詞を使う。形容詞より、店、坂道、市場、季節、移動手段。

## 語彙の統一

| 表記                    | 用途                                                                   |
| ----------------------- | ---------------------------------------------------------------------- |
| 書き手                  | 記事を書く人。普段はこれ。                                             |
| 現地ライター            | プロ感を出したいとき限定。多用しない。                                 |
| クリエイター            | UI のラベルや権利系（収益分配）でだけ使う。本文では「書き手」。        |
| 在外邦人                | 採用文脈で位置づけを伝えるときだけ。普通の本文では「現地で暮らす日本人」 |
| 観光ガイドにはない       | 多用すると安っぽい。1 ページに 1 回まで。                              |
| ローカル / ローカル度    | データ用語として残すが、本文では「地元」「現地の」                     |
| マルシェ / デモ / 工事   | OK                                                                     |
| 殺虫剤、納税、こども連れ | 駐在者情報の具体性。覚えておく。                                       |

## 「Coming Soon」「準備中」

- カードのバッジ: `準備中`（短く）
- 国詳細などのセクション見出し: `この街で書ける人を、探しています`
- Coming Soon 国の説明: `この国に住んでいて、街のことを書いてみたい方を探しています`
- 「順次公開します」は使う。「ご期待ください」は使わない。

## 各画面の現状コピー一覧

### `/`（ホーム）

- Hero kicker: `その街に、暮らしている人から`
- Hero h1: `旅は、誰の言葉で読むかで変わる。`
- Hero subtitle:  
  `Locore は、世界の街に暮らす日本人の書き手が、自分の毎日のなかで取材し、編集して届ける小さな旅行誌です。SNS では流れていってしまう「現地のこと」を、後から読み返せる短い物語の形にしました。`
- 国カルーセル: `行き先から探す / どこの暮らしを覗きますか？`
- スポット紹介: `スポット紹介 / 一軒の店、一本の坂道。`
- 旅程プラン: `旅程プラン / 現地民の半日、1 日の歩き方`
- 駐在者情報: `駐在者情報 / 暮らしの実務、ぜんぶ`

### `/region/[slug]`

- 新着ニュース: `新着ニュース / {region} の、今日と明日`
- ロック画面の説明: `この街で書ける人を、いま探しています。`

### `/country/[code]`

- 公開中の地域: `公開中の地域 / この街には、書き手がいます`
- 準備中の地域: `準備中の地域 / この街で書ける人を、探しています`
- Coming Soon 国: `{country}は、もうすぐ`

### `/world`

- h1: `旅したい国を、ぜんぶ。`
- subtitle: `いま読める国は N カ国。残りの国も、現地に書ける人を見つけた順に開けていきます。気長に育てていく予定です。`

### `/board`

- h1: `パリの、今日と明日。`
- 空状態: `今朝はまだ何も書かれていません。毎朝、現地時間の 7 時前後に更新します。`

### `/search`

- h1: `書いてある言葉から、辿る`
- 入力前: `検索したい言葉を入力してみてください。`
- 0 件: `その言葉ではまだ書かれていません。`

### `/library`

- h1: `あとで読む / 行く`
- subtitle: `気になった記事・旅程・スポットを、自分の旅の前夜に見返せるようにしておく場所。`

### Paywall

- 見出し: `ここから先は、書き手のサポート`
- 説明: `購入すると本文の続き、スポット名、住所、地図の正確な位置がすべて開きます。支払いの 70% は書き手にお渡しします。`
- CTA: `¥XXX で読み進める`

### Founders バナー（region ホーム）

- h2: `最初の 50 人を、ぼくらは長く覚えていたい。`
- 説明: 信頼で成り立つ小さなメディア / 収益分配の優遇 / 永久バッジ / 発言権 / フォロワー数より街への愛着の深さ

### `/about-service`

完全に編集トーンで本番化済み。ブロック構成:

1. ガイドブックには載らないものを、信頼できる人から
2. 何を読めるのか（3 種類の記事）
3. どうやって「現地民の本物」を担保するか
4. 読み手のため、書き手のため
5. 広げ方
6. お金のこと（70% 還元、Founders 優遇）
7. 運営について（個人事業主、Stripe）

## トースト / 操作系

| 操作          | 文言                                                                |
| ------------- | ------------------------------------------------------------------- |
| 旅程を保存    | `旅程を保存しました` / アクション「ライブラリを開く」               |
| 旅程を外す    | `保存ライブラリから外しました`                                      |
| スポット保存  | `「（フォルダ名）」に追加しました`                                  |
| 一括スポット  | `この記事の N 箇所をまとめて保存します。`                           |
| 購入完了      | `購入完了：本文とスポットが解放されました`（既存）→ 後ほど「開きました」に統一予定 |
| 未ログイン    | `ログインすると保存できます` / アクション「ログインする」           |

## 今後さわるとき

- 同じ画面で「現地民」「現地クリエイター」「在外邦人」を 2 つ以上混ぜない
- 数字は具体的に出す（「数十カ国」より「28 ヶ国」）
- 「！」は 1 ページに 1 個までを目安
- emoji は本文・タイトルでは使わない（カードの装飾 emoji 等は別途整理予定）
- 「クリエイターダッシュボード」など SaaS 業界用語を、「売上レポート」「あとで読む / 行く」のように行動ベースに言い換える方向で

## 参考

- Lonely Planet: 落ち着いた一人称、編集記事の文体
- monocle.com: ニュース + ライフスタイル誌の口調

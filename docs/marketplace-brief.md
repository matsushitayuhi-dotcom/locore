# 海外在住日本人向け スキル／ネットワーク マーケットプレイス — 事業・プロダクトブリーフ

> このドキュメントは Claude Code に渡す設計コンテキスト。事業の意思決定と、それが含意するプロダクト要件をまとめたもの。リポジトリ直下（例: `docs/brief.md`）か `CLAUDE.md` から参照させる想定。

---

## 1. コンセプト

- **一行**: 海外在住の日本人（駐在員・留学生）が、自分の人脈・知識・サービスを売り、運営は取引手数料を取るマーケットプレイス。
- **売り手**: 現地在住の日本人。**買い手**: ①日本側の個人/企業 ②現地を訪れる旅行者 ③同じ現地の日本人 ④B2B（日本企業）。
- **コンテンツ（記事・ポートフォリオ）の役割**: 商品ではなく、売り手の**集客・ブランディングの窓口**。本人が自分のために書く → 原稿料ゼロでコンテンツが集まる。発見の入口であり、中抜きされにくい上流。
- **ローンチ都市**: パリ（理由は §5）。最初は1都市集中。

---

## 2. 収益モデル

- **取引手数料（テイクレート）を最初の軸に**。登録無料・成約時のみ課金。供給（出品者）を集める初動に最適。
- **料率の目安: 10〜15%**。高単価・リピート・信頼関係型なので、高すぎると中抜きを自分で煽る。
- **必須前提**: 手数料を取るには**お金の流れを自社で握る**こと。予約＋決済＋エスクローを自プラットフォーム上で完結させる。発見だけ提供して取引が当事者間で閉じると1円も取れない。
- **将来の拡張**: 価値が証明できたら、上位表示・優先サポート等の有料オプションや出品者向け月額プランを後乗せ。手数料 first はあくまで順番の問題。

## 3. 提供価値（出品カテゴリ）

### A. 外向き（買い手=日本側／旅行者／B2B）
- 買付・調達・代理購入（古着・ヴィンテージ、ワイン、限定品、食材、OEM工場探し）
- アテンド・現地同行（観光、商談＋通訳、展示会視察、物件内見代理）
- 撮影（ロケフォト、物撮り、店舗/不動産、SNS素材）
- 通訳・翻訳・代理交渉
- リサーチ（市場・競合・棚調査、エリア/学校情報、規制のリアル）
- 専門知識コンサル（業界知見、留学/進学、現地就職、法人設立、不動産投資）
- ロジ・実務代行（転送・発送、検品梱包、難予約・チケット代行）

### B. 現地日本人同士（買い手=現地在住者・新参）
- 帰国者の家具/家電/車の譲渡・売買
- ベビーシッター・託児・送迎
- 家庭教師・補習・帰国子女受験
- 出張ヘアカット・美容
- 手作り料理・日本食販売
- 各種レッスン（ピアノ・書道・そろばん・子の日本語維持）
- 便利屋（引っ越し手伝い、運転、組み立て）
- 七五三・家族写真撮影
- 確定申告・日本＋現地税務サポート
- 病院/役所への同行サポート

### C. アクセス・コーディネート（パリ特化・プレミアム）
- ファッション（ショールーム、ヴァント・プリヴェ、Fashion Week、アトリエ、バイヤーコネ）
- ワイン（ドメーヌ直、カーヴィスト、試飲会）
- ガストロノミー（予約困難店、シェフ/パティシエ紹介、市場ツアー）
- アート（ヴェルニサージュ、アトリエ訪問、アートフェア）
- **言葉の設計上の注意**: 「招待の転売」は会員制クラブ等で譲渡禁止のことが多い。**「案内・同行・コーディネート」**（本人の関係で連れて入る）と位置づけ、グレーを避けつつ同じ価値を売る。

## 4. 二層戦略

| 層 | 役割 | 特徴 | カテゴリ |
|---|---|---|---|
| 流動性エンジン | 人と取引を日々回す | 小口・高頻度・手数料薄 | 買付/発送代行、現地日本人同士のサービス |
| 収益の柱 | 稼ぐ | 大口・低頻度・高単価 | アクセス・コーディネート、通訳/B2Bリサーチ |

前者で人を集めてレビュー・習慣を貯め、後者で稼ぐ。相互補完。

---

## 5. ローンチの楔（パリ）

- **最初の看板候補 = 買付・引き取り・発送代行**。日本側（BUYMA等のバイヤー）の需要が既に活発で、今は無料掲示板で雑に回っている。
- **既存掲示板の状況**: パリ掲示板、tomoslist、パリナビ、GojoComyu 等が存在。いずれも**無料の掲示板＝発見だけ**で、決済・エスクロー・レビュー・手数料がない（パリナビは募集/宣伝投稿自体を禁止）。
- **差別化の楔**: Stripe決済＋代金預かり＋実績レビュー。需要は実証済み、欠けている「取引・信頼レイヤー」を埋める。
- パリを選ぶ理由: 街の経済が「ドアを知る人にしか入れない」領域（ファッション/ワイン/食/アート）＝アクセス商品が映える。高スキル・好人脈の日本人コミュニティの厚みもある。

## 6. 需要の見極めフィルター

候補カテゴリを評価する4軸（多く当たるほど有望）:
1. 繰り返し発生するか
2. 痛みが大きいか（失敗時の損失・手間が大きい＝財布が緩む）
3. DIYしにくいか（現地/語学がないと無理＝代替不能）
4. 既にお金が動いているか（需要の証明）

## 7. 中抜き（disintermediation）対策

**原則**: ゼロにはできない。締め付けすぎると人が離れる。勝ち筋は「留まるのが得」＋「構造的に抜けにくい領域に収益を寄せる」。中抜きは一定率許容する税金と捉える。

### 囲い込み（カロリー高い順）
- **オンライン上にだけ実績・レビューが貯まる**（最強。外に出る＝信用ゼロから）
- エスクロー＋トラブル仲裁（双方の保険）
- 新規顧客の継続流入（リピートは直接行かれても新規の蛇口は奪えない）
- 買い手側も縛る（買い手が代金保護を求めれば直接取引を拒否してくれる）

### 価格設計
- 料率は低め（10〜15%）
- **同一客との2回目以降は料率を下げる/上限を設ける**（中抜きはリピート関係に集中するため、これが地味に最も効く）
- 高頻度の売り手は月額プランで実質定額化

### 摩擦（最小限・抑止程度）
- 予約/決済確定まで連絡先（電話・メール・本名）を伏せる
- メッセージ内の連絡先・「直接やりませんか」を検知して警告
- 規約で迂回取引を禁止＋違反でアカウント停止
- ※「監視されてる感」はコミュニティ型と相性最悪。やりすぎ厳禁。

### 層別の方針
- **買付・アクセス層**: エスクロー＋実績で構造的に守る（越境・他人間はエスクローが取引成立の理由そのもの＝抜けにくい）
- **現地反復サービス層**: 直接取引にほぼ流れる前提で割り切る。手数料は「初回マッチング」で取り深追いしない。収益の柱ではなく流動性エンジンと位置づける。

## 8. 信頼・運用

- レビュー/実績（オンライン専用に蓄積）
- 本人確認・出品者の簡易審査
- エスクロー: 代金保留 → サービス完了で解放 → 紛争時は返金/仲裁
- メッセージング（連絡先マスキング込み）

---

## 9. リスク・法務メモ

- **中抜き**（§7で対処）
- **規制のグレー**:
  - 有償ガイド/アテンド → 国によりガイド資格
  - 古着等の買付・転売 → 輸入・関税・古物営業・ブランドの転売規約
  - ワイン → 酒類関連
  - アクセス系 → 会員制/招待の譲渡禁止に注意（「コーディネート」に寄せる）
- **規約での線引き**: プラットフォームは仲介者、取引責任は当事者、という整理を最初に固める。

## 10. プロダクト要件への示唆（実装メモ）

戦略 → 必要機能のマッピング。Claude Code 側の実装の出発点。

- **決済/エスクロー**: Stripe Connect（開設済み）。マーケットプレイス型で、代金を一旦保留 → 完了で出品者へ分配（手数料控除）→ 紛争で返金。返金/チャージバック/紛争フローを設計。
- **手数料エンジン**: 料率テーブルを持つ（カテゴリ別・初回/リピート逓減・絶対額上限）。同一買い手×売り手のリピート判定が必要。
- **レビュー/評判**: 取引に紐づくレビュー。プラットフォーム外に持ち出せない設計（=囲い込みの主軸）。
- **メッセージング**: アプリ内チャット。予約確定まで連絡先マスキング。連絡先/迂回誘導の検知（正規表現＋警告）。
- **出品・検索**: カテゴリ（§3）、職種ごとに発信フォーマット可変（記事/フォトエッセイ/作品集/プロフィール）。
- **コンテンツ → 取引導線**: 記事/ポートフォリオ（無料・集客用）から、同一プラットフォーム上の予約・決済へ自然に流す。
- **本人確認・審査**: 出品者オンボーディングに簡易KYC。
- **多通貨/越境**: 日本側買い手 ↔ パリ売り手。EUR/JPY、為替表示、発送追跡。
- **創業メンバー施策**: 最初のN人/N取引は手数料ゼロor半額 → 後で通常料率（料率テーブルで期間/回数制御）。

## 0. ドキュメント概要

本書は Locore（ロコレ）Phase 1 (MVP) の UI/UX デザインガイドラインおよび主要画面ワイヤーフレームを定義する。Figma 起こし、および shadcn/ui ベースの実装に直接落とし込める粒度で記述する。

設計原則：

1. **静かに豊かに**：余白を恐れず、情報密度よりも一目の心地よさを優先
2. **没入感の演出**：カバー写真と本文タイポで「読み物としての価値」を可視化
3. **マップとフィードの双子設計**：どちらの世界にいても文脈を失わない
4. **書き手の権威を可視化**：認証バッジ・居住歴を信頼性の核に据える
5. **モバイル7：デスクトップ3**：旅行中の片手操作を最優先動線に

## 1. デザイン原則とブランドトーン

### 1.1 デザイン原則

| 原則 | 意味 | 反例 |
|------|------|------|
| **Quiet Premium** | 控えめなプレミアム感。金箔・グラデではなく、紙のような白とインクのような黒で語る | ネオン色、ゴールドのアクセント、過剰なグラデーション |
| **Editorial First** | 雑誌・書籍のような誌面構成。タイポグラフィの階層が主役 | 飾り罫、装飾アイコン、ステッカー風UI |
| **Map as Stage** | マップは情報表示装置ではなく舞台。記事の景色を描く | 純正Google MapsのUIをそのまま使う |
| **Show, Don't Tell** | 「映え禁止」は文字で書かず、UIの余白と被写体選定で示す | 「映えNG」の警告ポップアップ多用 |
| **Earned Trust** | バッジ・居住歴は控えめに、しかし常に視野に入る位置に | バッジを巨大化、星マーク多用 |

### 1.2 ブランドキーワード

```
Calm  /  Curated  /  Inked  /  Slow  /  Insider
静か   厳選       インク調   ゆったり  内側の視点
```

### 1.3 「映え禁止」のUIでの表現方法

- カバー画像のアスペクト比を **16:9 ではなく 3:2**（写真集・雑誌の比率）に統一し、Instagram的な正方形を排除
- 評価軸を「ローカル度」が先、「満足度（星）」が後
- 「人気急上昇」「バズり」のような煽り表現を一切使わない
- カードシャドウは最小限、ボーダーで構造を表現する

### 2.2 タイポグラフィ

#### 2.2.1 フォントファミリー

| 種別 | フォント | フォールバック | 用途 |
|------|---------|--------------|------|
| 日本語 見出し・本文 | **Noto Serif JP** (Regular 400 / Medium 500 / SemiBold 600) | "Yu Mincho", "游明朝", serif | 本文・見出しは原則明朝で統一し、誌面感を出す |
| 日本語 UI | **Noto Sans JP** (Regular 400 / Medium 500 / Bold 700) | "Hiragino Sans", "Yu Gothic", sans-serif | ボタン、ラベル、フォーム、ナビ |
| 欧文 見出し | **Fraunces** (Variable, optical size) | Georgia, serif | ロゴ、欧文タイトル、特集 |

#### 2.2.3 タイポルール

- 行長：本文は **42〜56字（日本語）** を最大。広い PC でも `max-width: 680px` を死守
- 段落間：本文 `1em + 8px` のマージン
- 見出し→本文間：見出しの 0.5em 上、1em 下
- 数値表記：金額は `mono-md`、`¥1,200` のようにスペースなし
- 句読点詰め：`text-spacing: ideograph-numeric ideograph-alpha`（CSS Text 4 対応時）

#### 3.1.2 要素仕様

**Header**
- 高さ 56px、`neutral-25` 背景、下端 `border-hairline`
- ロゴ：左寄せ、`Fraunces SemiBold 22px`、`neutral-900`、左マージン 16px

- スクロール時：背景の透明度を変えず、ボーダーのみ強調

**Crisis Banner**
- 条件：ユーザーの予定旅程または閲覧都市にイベントがある場合のみ
- 高さ 48px、`warning-50` 背景、左 4px の `warning-500` アクセントバー
- アイコン `AlertTriangle` 16px + テキスト `body-sm` `warning-700`
- 右端に `X` アイコン（dismiss、24時間サイレンス）

**Filter Bar**
- 高さ 52px、`neutral-25` 背景、`position: sticky; top: 0`
- チップ群：`radius-full`、`neutral-100` 背景、選択時 `primary-700` 背景＋`neutral-0` テキスト
- チップサイズ：高さ 36px、横パディング 14px、`caption` サイズ
- 右端の `SlidersHorizontal` アイコン：詳細フィルタモーダルを開く

**View Toggle**
- セグメントコントロール、`neutral-50` 背景、選択側 `neutral-0` 背景 + `shadow-xs`
- 高さ 40px、横幅 240px 中央寄せ

**Featured Strip**
- 高さ 120px、横スクロール（snap-x）
- セクションラベル：`overline` `neutral-500`、`―― EDITOR'S PICK ――` のように罫線で囲む
- カード：240×100、左右ガター 12px

## 4. インタラクション・モーション設計

### 4.1 モーション原則

- **Easing**：標準は `cubic-bezier(0.4, 0.0, 0.2, 1)`（ease-out 寄り）
- **Duration**：
  - Micro：100-150ms（チップ、ボタンの色変化）
  - Standard：200-300ms（ホバー、モーダル開閉）
  - Macro：400-500ms（ページ遷移、ボトムシート）
- **Reduce Motion**：`prefers-reduced-motion: reduce` 時は全アニメーションを 1ms に短縮

### 4.2 記事カードのホバー

PC のみ：

```
通常:
  shadow-xs / image scale 1.0 / title color neutral-900

ホバー (200ms ease-out):
  shadow-sm
  image transform: scale(1.02)
  title color neutral-900 (no change, already strong)
  「+ 旅程に追加」ボタンのアイコンが Plus → Sparkles に変化（150ms）
```

タップ（モバイル）：押下中 `opacity: 0.92` 100ms

### 4.3 マップピン展開

```
通常: 28×28 円
ホバー (PC):
  scale 1.15, shadow-md, 200ms
クラスタピン展開 (タップ):
  ピン中央から sub-pins が 200ms で半径方向にスプレー
  framer-motion: stagger 30ms / pin
ボトムシート出現:
  off-screen (translateY 100%) → peek (translateY calc(100% - 120px))
  300ms ease-out
```

### 4.4 トリッププランナー DnD

```
ドラッグ開始 (300ms long-press on mobile):
  haptic feedback (タップ振動)
  ドラッグ中要素: scale 1.02, shadow-lg, opacity 0.95, rotate 1deg
  他アイテム: 順次ずれる、200ms ease-out

ドロップ可能ゾーン:
  primary-100 background dashed border 2px (10px dash, 4px gap)
  drop indicator pulses 600ms loop

ドロップ完了:
  150ms scale 0.98 → 1.0 settle
  toast notification 「移動しました」 (bottom, 2s, fadeout)

別 Day へドラッグ:
  Day タブを 600ms hover で switch
  switch時 軽いハイライトフラッシュ
```

## 6. アクセシビリティ

### 6.1 WCAG 2.1 AA 準拠

#### 6.1.1 コントラスト比

全テキスト・アイコンは AA を満たす（4.5:1 / 3:1）。検証済みペア：

| FG / BG | 比 | 用途 |
|--------|----|----|
| `neutral-900` / `neutral-25` | 13.8:1 | 本文・見出し ✓ |
| `neutral-700` / `neutral-25` | 8.9:1 | 本文 ✓ |
| `neutral-500` / `neutral-25` | 4.6:1 | キャプション ✓ |
| `neutral-400` / `neutral-25` | 3.2:1 | サブテキスト（大サイズのみ AA、小サイズは AA未満。注意）|
| `primary-700` / `neutral-25` | 8.5:1 | リンク・ボタンテキスト ✓ |
| `neutral-0` / `primary-700` | 8.7:1 | プライマリボタン上の白文字 ✓ |
| `accent-500` / `neutral-25` | 5.4:1 | バッジ・進捗バー ✓ |
| `secondary-500` / `neutral-25` | 4.7:1 | 特集ラベル ✓ |
| `danger-500` / `neutral-25` | 5.0:1 | エラーテキスト ✓ |
| `warning-500` / `neutral-25` | 3.6:1 | 大サイズのみ AA、小は AA未満 → 警告メッセージは 14px 以上 |

`neutral-400` と `warning-500` は使用文脈を制限する。

#### 6.1.2 フォーカス可視化

- 全インタラクティブ要素に明確なフォーカスリング
- スタイル：`outline: 2px solid #26356B; outline-offset: 2px;`（`primary-700`）
- ボタン・リンク・入力・カードに統一適用
- カードのフォーカスはカード全体にリング、内部要素にはリング無し（タブ移動の混乱を回避）

#### 6.1.3 キーボード操作

- すべての機能をキーボードで完遂可能
- フォーカス順序は視覚的順序と一致
- モーダル：`Esc` で閉じる、フォーカストラップ
- ボトムシート：`Esc` で peek → 閉じる、`Tab` でシート内移動
- ドラッグ&ドロップ：`Space` で持つ、矢印キーで移動、`Space` で置く（@dnd-kit/sortable のキーボードコンディション準拠）

#### 6.1.4 スクリーンリーダー対応

- セマンティック HTML 優先（`<main>`, `<nav>`, `<article>`, `<aside>`, `<button>`）
- `aria-label`：アイコンのみのボタン
- `aria-live="polite"`：トースト、AI ヒント
- `aria-live="assertive"`：エラー、クライシス情報
- `aria-current="page"`：ナビの現在ページ
- 進捗バー：`role="progressbar"` + `aria-valuenow / aria-valuemin / aria-valuemax`
- ローカル度バー：`<div role="meter" aria-label="ローカル度" aria-valuenow="65" aria-valuemin="0" aria-valuemax="100">` + 視覚的バー
- マップ：`role="application"` + 代替リスト表示（マップビューと同じ結果のリスト）

#### 6.1.5 代替テキスト

- 全画像に `alt` 属性
- 装飾画像：`alt=""`
- カバー画像：記事タイトルを alt
- スポット写真：「{店舗名} の店内写真」のような説明
- 著者アバター：「{著者名} のプロフィール写真」

#### 6.1.6 動画・モーション

- 自動再生は禁止（特に SNS 埋め込み動画）
- `prefers-reduced-motion` でアニメーション無効化
- 点滅は 3Hz 以下

#### 6.1.7 色のみに依存しない情報伝達

- ローカル度ピン：色 + アイコン形状の差異
- ヒートマップ：色 + テキスト凡例
- レビュー満足度：色 + ★ アイコン
- フォームエラー：赤色 + アイコン + テキスト

#### 6.1.8 言語属性

- ルート HTML：`lang="ja"`
- 欧文混入箇所：`<span lang="en">Editor's Pick</span>`
- 著者名（外国人ライター名等）：適宜 `lang` 属性

#### 6.1.9 ズーム対応

- `viewport`: `width=device-width, initial-scale=1`（user-scalable は禁止しない）
- 200% ズーム時もコンテンツが横スクロールなく読める（`max-width: 720px` の本文は問題なし）

### 6.2 認証・購入特有の配慮

- パスワード入力：表示/非表示トグル
- 決済フォーム：Stripe Payment Element の標準アクセシビリティに準拠
- エラー：`aria-describedby` で入力フィールドと紐付け

### 6.3 モバイル特有

- セーフエリア：`env(safe-area-inset-*)` 使用
- 横向き対応：マップ・本文は両対応、エディタは縦推奨（横はスナックバーで案内）

## 9. コピーライティングのトーン

UI 文言の方針：

### 9.1 基本ルール

- 敬体（です・ます調）を基本とし、UI ラベルは体言止めや命令形を許容
- 「ユーザー様」のような過剰敬語は使わない
- 一人称「わたしたち」は最小限
- 「素敵な」「魅力的な」のような形容詞を避け、具体名詞で語る
- 略語：「Maps」「IG」「YT」のような略称は本文では使わず、UI チップやアイコンラベルでのみ可

### 9.2 主要文言（参考）

| 場面 | 文言 |
|-----|-----|
| トップ記事「公開」ボタン | 「公開申請する」（「投稿する」より重さがある） |
| 購入ボタン | 「¥800 を支払う」（金額を含めて確実性を伝える） |
| カード CTA | 「旅程に追加」（「保存」より能動的）|
| 空状態（フィード） | 「まだ記事がありません。/ もうすぐ最初の記事が届きます」 |
| エラー（決済） | 「決済を完了できませんでした。カード情報をご確認ください」 |
| 認証完了 | 「申請を受け付けました。/ 審査結果はメールでお知らせします」|
| クライシス警告 | 「⚠ パリ：5/3 RATP 部分ストライキ予定 / 詳細を見る →」 |
| マップビューラベル | 「ヒートマップ」「現在地」（よけいな説明文を付けない）|

### 9.3 禁止表現

- 「神」「最強」「絶対」「完璧」「世界一」（PRD §10.3）
- 「映え」「バズる」「リール風」「インスタ映え」
- 「ぜひ」「必見」「外せない」（広告的）

## 12. オープンクエスチョン

実装着手前に決定が必要：

1. **本文フォントの最終確定**：Noto Serif JP の「明朝で全文」案は読みやすさのテストが必要。Noto Sans JP との 50/50 ABテストを推奨
2. **ダークモード**：Phase 1 では未実装。トークンは将来追加可能な構造で命名済み
3. **マップピンの「写真サムネ版」**：PRD §6.8.2 で「ピンは記事サムネ or テーマ別アイコン」とあるが、本書ではテーマ別アイコンに寄せた。サムネ版はズームレベル別の出し分け要検討
4. **特集の縦長ヒーロー比率**：本書では 16:9 だが、雑誌的には 4:5 縦長も候補。A/B 推奨
5. **Bottom Nav の中央 + ボタン**：「+ 旅程」を中央配置するか、Plus FAB を別途置くか
6. **ローカル度バーの離散表示 vs 連続表示**：●●●●●●○○○○ のような 10 段表示と、滑らかなバーのどちらを正とするか統一する。本書は混在気味のため Figma で要決定
7. **書き手記事の「ファウンディングクリエイター」バッジ表現**：永久バッジの視覚デザインが未定（Star, Crown, Anchor 等）
